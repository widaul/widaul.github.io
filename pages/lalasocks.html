Project Overview

Scalable backend services for an e-commerce platform.

LalaSocks is a full-stack e-commerce platform developed for a local socks retailer to support online shopping and simplify store operations. As the Backend Developer, I built the server-side infrastructure, including RESTful APIs, JWT authentication, database management, and business logic to power product management, customer transactions, and order processing.

Responsible for developing backend modules, database design, authentication, and business logic using node.js Hapi.

Problem Statement

Challenges Before the System Was Developed

Customers primarily purchased products through third-party marketplaces, limiting the store's ability to provide a dedicated shopping experience and strengthen direct customer engagement.

Limited Online Presence

The business relied on external marketplace platforms, making it difficult to establish its own brand identity and shopping experience.

Product Discovery

Customers had limited ways to browse products through a dedicated catalog with complete product information and categories.

Shopping Experience

The absence of a dedicated e-commerce website reduced convenience for customers when exploring products, managing carts, and completing purchases

Customer Engagement

The store had limited opportunities to communicate promotions, featured products, and updates directly to customers.

Proposed Solution

Dedicated E-Commerce Platform

LalaSocks was developed as a web-based e-commerce platform that provides customers with a convenient and seamless online shopping experience while strengthening the store's digital presence.

Product Catalog

Provides an organized catalog where customers can browse products with detailed descriptions, pricing, and available sizes.

Shopping Cart

Allows customers to save selected products before completing their purchases.

Secure Checkout

Enables customers to place orders through a simple and secure checkout process using midtrans

User Accounts

Supports customer registration, login, and profile management with JWT-based authentication.

Responsive Experience

Delivers a responsive interface that allows customers to shop comfortably from both desktop and mobile devices.

System Design

System Architecture

The backend is built using Node.js and Hapi.js following a RESTful architecture. Incoming requests are authenticated with JWT before being routed to business logic services for product catalog, checkout, and order processing. Data is persisted in MySQL, while payment transactions are securely integrated with the Midtrans Sandbox payment gateway to simulate real-world online payment workflows.

Database Structure

Main Database Tables

Users

Stores user account information, authentication credentials, and profile details required for secure access to the e-commerce platform.

Cart

Stores products selected by users before checkout, including product quantities and cart information for each customer.

Orders

Stores customer orders created during checkout, including order status, payment information, and transaction details generated through Midtrans Sandbox integration.

Order Items

Stores individual products associated with each order, including product details, quantities, and prices, enabling accurate order processing

Security

Authentication Flow

The backend uses JWT-based authentication to validate user credentials and protect API endpoints. Upon successful login, users receive an access token that is used to authorize all subsequent requests to the backend services.

Backend

REST API Endpoints

The backend exposes a collection of RESTful APIs that power the LalaSocks e-commerce platform. These endpoints handle user authentication, shopping cart operations, order processing, and product management, enabling seamless communication between the frontend and backend through structured JSON responses.

Method Endpoint Description
POST /register Register a new customer account.
POST /user/login Authenticate users and generate a JWT access token.
GET /profile Retrieve the authenticated user's profile information.
GET /allProduct Retrieve the complete product catalog.
POST /keranjang Add products to the shopping cart.
GET /keranjang Retrieve the authenticated user's shopping cart.
POST /keranjang/update Update product quantities in the shopping cart.
POST /pesanan/manual Create a new order and initiate the checkout process.
POST /payments/callback Receive payment status notifications from Midtrans Sandbox.
GET /pesanan Retrieve the authenticated user's order history.
GET /admin/pesanan Retrieve all customer orders for administrative management.
PUT /admin/pesanan/{order_id} Update the status of a customer order.

Reflection

Lessons Learned

API Design

Designing modular RESTful APIs improves maintainability and simplifies frontend-backend integration.

Secure Authentication

WT-based authentication provides a lightweight and secure approach to protecting API endpoints and managing user sessions.

Payment Gateway Integration

Integrating Midtrans strengthened my understanding of online payment workflows, asynchronous callbacks, and transaction lifecycle management.

Backend Architecture

Structuring business logic separately from routing and database operations results in cleaner, more scalable backend applications.
