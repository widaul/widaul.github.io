Problem Statement

Challenges Before the System Was Developed

Small-scale fish farmers often rely on manual observations to monitor water quality, estimate harvest time, and determine feeding schedules. Limited access to affordable digital technologies makes decision-making less efficient and can reduce productivity.

Water Quality Monitoring

Farmers manually assess water conditions based on experience, making it difficult to detect changes accurately.

Harvest Prediction

Harvest timing is estimated manually, increasing the risk of harvesting too early or too late.

Feed Management

Feed requirements are calculated without data-driven predictions, leading to overfeeding or unnecessary costs.

Limited Access to Technology

Existing smart aquaculture solutions often require expensive IoT devices or complex systems that are not affordable for small-scale farmers.

Proposed Solution

Intelligent Fisheries Management Platform

To address these challenges, AquaSmart Fisheries was developed as an affordable web-based platform that leverages machine learning to help fish farmers make better decisions without requiring expensive hardware.

Water Quality Prediction

Farmers manually assess water conditions based on experience, making it difficult to detect changes accurately.

Harvest & Feed Prediction

Linear regression predicts the optimal harvest time and recommended feed requirements to improve farming efficiency.

Personalized Recommendations

The system provides actionable recommendations based on prediction results, helping farmers maintain healthy pond conditions.

Accessible Mobile Application

Built for Android, AquaSmart Fisheries allows farmers to access predictions, recommendations, and farming information anytime through their smartphones, making smart aquaculture more affordable and accessible.

System Design

System Architecture

The application follows a cloud-based architecture where users interact with the Android application, API requests are processed by Node.js services deployed on Google Cloud Compute Engine, machine learning models generate prediction results, and all application data is stored in Cloud SQL. Machine learning model files are maintained in Cloud Storage, enabling scalable and reliable prediction services for water quality, harvest estimation, and feed recommendations.

Database Structure

Main Database Tables

Users

Stores user account information, authentication credentials, profile details, and profile picture references.

Harvest Predictions

Stores fish farming data submitted by users, including fish type, fish size, water condition, predicted harvest time, and recommended feed amount.

Water Quality Input

Stores water quality parameters entered by users, such as pH, temperature, and turbidity, which are used as input for the machine learning model.

Water Quality Output

Stores prediction results generated by the machine learning service, including water quality classification and recommended actions.

Security

Authentication Flow

The backend uses JWT-based authentication to validate user credentials and protect API endpoints. Upon successful login, users receive an access token that is used to authorize all subsequent requests to the backend services.

REST API Endpoints

A collection of RESTful endpoints was developed to support data submission, prediction processing, retrieval of prediction history, and resource management for the AquaSmart Fisheries mobile application.

Method Endpoint Description
GET /api/water-quality Retrieve submitted water quality records
POST /api/water-quality Submit water quality parameters for prediction.
GET /api/water-quality/predictions Retrieve water quality prediction history
DELETE /api/water-quality/{id} Delete a specific water quality record.
POST /prediksiPanen Submit harvest prediction request.
GET /prediksiPanen Retrieve harvest prediction history.
DELETE /prediksiPanen/{id} Delete a harvest prediction record.
